## Analysis Template & Auto-Creation

- **Preferred analysis template path:** `templates/analysis_template.md`
- **Secondary (fallback) path:** `analysis/analysis_template.md`
- **Final fallback:** use the embedded default template (below) if neither file exists.

### Behavior
1) When asked to create a new analysis file:
   - **Fetch latest** from repo first (to avoid overwriting).
   - Try to load `templates/analysis_template.md`. If not found, try `analysis/analysis_template.md`. If neither is found, use the embedded template.
   - Fill in front matter fields (`title`, `linked_case_id`, `created`, `last_updated`, `tags`, `sources`).
   - If the analysis links to a known case, set `linked_case_id` to the case’s stable ID or filename stem.
   - Write to `/analysis` using the naming:
     - If tied to a case: `<case-id>-analysis.md` or `<case-filename-without-ext>-analysis.md`
     - Otherwise: `YYYYMMDD-<short-topic-slug>-analysis.md`
   - **Commit directly** to `main` with:
     - New: `Add analysis: <Title> (via Investigative GPT)`
     - Update: `Update analysis: <Title> (via Investigative GPT)`
   - Upsert `/analysis/_index.json` (newest first).

2) Always **pull latest**, merge, and only then commit. If merge ambiguity arises, preserve both sections in the body and clearly annotate.
